# 简单讲解

![w:600](https://pic1.zhimg.com/v2-e230a5a6679bed4d5a24e9aa783a0b5c_r.jpg  )

大家都学过分治吧.我的理解线段树本质上就是一种分治结构,这个分治结构把任意序列上的问题分成$\log$段,从而支持我们快速找到任意一个区间的答案,或者把一个对任意区间的操作摊到$\log$个段上.由于我们要把一堆段的答案拼起来,所以信息要求可以合并.

---

# 栗子

- 单点加
- 区间求和
- $n,q \le 10^6$

---

# Solution

好像直接秒.对每个节点维护所管辖区间的和.修改的时候覆盖一个点的有$\log$个区间,暴力把它们的改一改,而查询就是把$\log$个区间的和直接加起来

---

# 栗子2(线段树1)

- 区间加
- 区间求和
- $n,q \le 10^6$

---

# Solution

被一个任意区间覆盖的区间个数是$O(n)$的,不能暴力改了.考虑我们只修改直接拆成的$\log$个,那么其他要改的区间都是它们的孩子.注意如果我们要访问一个点,我们一定会走过根到它的一条路径,所以我们给每个点记录它的儿子们应该再加多少,那么每次顺着根走下去的时候把这个该加多少给儿子加上,再给儿子也打上标记,复杂度就仍是1log.

这个技术就叫懒标记,能使用它的条件有两个:
1. 我们可以快速合并两个修改,比如本题区间先加$v1$,再加$v2$,就等价于一次修改区间加$v1+v2$,这就叫可合并,类似的还有区间赋值等.
2. 我们可以不用递归子树就确定这个修改对当前答案的影响.比如本题我们不需要看它的儿子就知道这个区间的答案增加了$delta*len$.

# 半睡半醒综合症(我起的)

大家有没有做过起床困难综合症啊

给一个序列,每个位置上有一个值和一个操作,操作是and,or,xor中的一个,每次选定一个区间,询问若一开始是一个$[1,m]$的数,把它依次执行这个区间的操作后走出来最大是多少

要求$n\log n$

---

# solution

首先按照起床困难综合症那题的套路,我们可以把每一位分开考虑然后从高位向低位贪心,问题变为如何求某一位进去啥能成1,我们考虑任意多个位运算复合起来都可以用进去0或1对应出来0还是1这样一个结构简单描述,而这个结构可以简单合并,于是用线段树维护这个东西,复杂度是$n\log n\log v$

然后再用位运算加速它,可以把一摞进去是0还是1压缩成1个二进制数,就结束了.

# 矩阵信息线段树

我听说你们都学过矩阵乘法优化dp啥的了.

这个其实说起来也简单,就是线段树维护的那个序列的元素是矩阵啊.

一般有两类,维护的是一摞矩阵加起来或者乘起来.

如果加起来那么还支持区间乘矩阵,但要是乘起来就只能赋值矩阵了.

来看几个栗子

---

# Fib

给一个序列$A$,支持:
- 区间加
- 询问$\sum_i fib_{a_i}$,$fib_i$表示斐波那契数列第$i$项.

---

# Solution

斐波那契的矩阵递推是众所周知的,那么就每个位置放一个向量$[f_{a_i-1},f_{a_i}]$,每次修改就是区间乘一个矩阵,你会惊喜的发现这玩意如果封装个结构体和裸的线段树写起来一模一样.

---

# 区间历史和

给一个序列$A$支持:

- 区间加
- 区间和
- 区间所有版本和

复杂度还是1log.

---

仍然是维护的矩阵和.考虑:

$$

\begin{bmatrix}
val & sum & 1\\
\end{bmatrix}
\begin{bmatrix}
1  & 1 & 0\\
0  & 1 & 0\\
add  & add & 1
\end{bmatrix}

$$

$sum$为历史和,$val$为元素值,$add$为增加的值.也可以把它拆成两个矩阵,一个用来加,一个用来更新历史和.

---

# 保卫半臂江山(是的我又来弱化了)

一个序列,每次修改一个位置,求不能选相邻两个的情况下的选的数的最大和.1log

---

不是学过矩阵优化dp了吗,来看:

$$
\begin{bmatrix}
f_{i-1,0}  &  f_{i-1,1}
\end{bmatrix}
\begin{bmatrix}
0  & a_i\\
0  & -inf
\end{bmatrix}
$$

用线段树维护这摞矩阵乘起来的值就好啦

# 栗子我也忘了是几

区间取min,区间最大最小值,区间和

---

区间取min比较麻烦,取min操作不满足第二条,我们不能通过不递归的快速知道取min的影响,考虑绕开它:

维护区间最大值$max$,次大值$second$,区间最大值个数$cnt$,则对$k$取min时:
1. $k \ge max$ 直接跳
2. $second \le k < max$ 只用修改$max$就好了,而且此时影响很好算
3. $k < second$ 往下递归,如果有一天,它们再相见,它将大于$second$

由于有第三种情况这个复杂度看起来很玄学,但是我来分析一下,每递归一次意味着取min完了后这个区间里不同的值的个数至少少了一个,所有区间里不同值的个数总和是$n \log n$的所以复杂度对了.
这个实现也有点细节,就是你需要维护最大值和其他值分别的加法标记,然后pushdown的过程还要分析一下这个区间的最大值在哪一边.代码等着线段树3一起看吧.

# 线段树3

再加上区间历史最值

怎么办啊

---

**维护历史值时我们要维护所有现在的值才能应对修改.** 因为你现在维护这么一摞值不就是为了应对修改吗.
于是我们一共维护:
- 最大值
- 次大值
- 最大值个数
- 普通数加标记
- 最大值加标记
- 历史最大值
- 历史次大值
- 历史最大值个数
- 历史普通数加标记
- 历史最大值加标记
- 区间和

# 代码

```cpp
/*
好难懂啊,解释一下变量名:
sum是和,sm是次大值,m是最大值
前面加个h是历史,加个l是标记,...ma是对最大值的,...na是普通值的
是不是很难理解,我来一个一个仔细讲讲
*/
void update(int u,int ama,int ahma,int ana,int ahna,int len){
    if(sm[u]!=-INF)sm[u]+=ana;
    sum[u]+=mcnt[u]*ama+(len-mcnt[u])*ana;


    bm[u]=max(bm[u],m[u]+ahma);
    lhma[u]=max(lhma[u],lma[u]+ahma);
    lhna[u]=max(lhna[u],lna[u]+ahna);

    lma[u]+=ama;
    lna[u]+=ana;
    m[u]+=ama;
}
```

```cpp
void pushdown(int u,int l,int r){
    int mid=(l+r)>>1;
    int maxu=max(m[u<<1],m[u<<1|1]);
    if(maxu==m[u<<1]){
        update(u<<1,lma[u],lhma[u],lna[u],lhna[u],mid-l+1);
    }else{
        update(u<<1,lna[u],lhna[u],lna[u],lhna[u],mid-l+1);
    }
    if(maxu==m[u<<1|1]){
        update(u<<1|1,lma[u],lhma[u],lna[u],lhna[u],r-mid);
    }else{
        update(u<<1|1,lna[u],lhna[u],lna[u],lhna[u],r-mid);
    }
    lma[u]=lna[u]=lhma[u]=lhna[u]=0;
}
```
